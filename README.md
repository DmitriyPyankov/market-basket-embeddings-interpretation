## Описание проекта  

Проект посвящён исследованию методов построения интерпретируемых представлений товаров (SKU) на основе данных рыночных корзин.  

Основная идея работы заключается не в построении классической рекомендательной системы, а в создании признакового пространства товаров, которое можно использовать в downstream-задачах:  

- рекомендации;  
- uplift-моделирование;  
- классификация товаров;  
- анализ ассортимента;  
- поиск связанных SKU;  
- построение дополнительных признаков для ML-моделей.  

В работе используются данные чеков розничной сети «Магнит». Каждый чек рассматривается как корзина товаров, а товар описывается через то, с какими другими товарами он встречается в реальных покупках.  
ATTENTION: Запустить проект возможно исключительно в инфраструктуре "Магнит", т.к. используются данные и внутренние библиотеки компании.   

---

## Основная идея

В проекте строятся два типа представлений SKU:  

1. **EASE-вектор**  

   Это вектор связей товара со всеми остальными товарами.  

   Каждая координата такого вектора соответствует связи с конкретным SKU.  
   Поэтому EASE-представление можно интерпретировать напрямую: можно посмотреть, с какими товарами данный SKU чаще всего связан по структуре чеков.  

2. **VAE-эмбеддинг**  

   Это плотное латентное представление товара, полученное с помощью вариационного автоэнкодера.  

   В отличие от EASE-вектора, VAE-эмбеддинг не интерпретируется напрямую по координатам, но позволяет учитывать более сложные скрытые зависимости между товарами и категориями.  

Итоговый подход объединяет эти два сигнала в гибридной модели.

## Что реализовано  

В проекте реализованы следующие компоненты:   

### 1. Word2Vec baseline  

Word2Vec используется как базовый подход.   

Корзина рассматривается как аналог предложения, а товары — как аналоги слов.    
Модель обучается на последовательностях товаров и позволяет получить начальные embedding товаров.  

Word2Vec используется только как baseline для сравнения.  

---

### 2. Модель EASE  

EASE используется как основная линейная модель связей между товарами.   

Модель обучается на sparse-матрице вида:  

```text
чек × товар
```

На выходе получается матрица весов:  

```text
B ∈ R^{N × N}
```

где каждая строка соответствует одному товару, а значения в строке показывают силу связи этого товара с другими SKU.  

EASE используется для:  

- построения интерпретируемого вектора товара;  
- генерации кандидатов для гибридной модели;  
- анализа совместного потребления.  

---

### 3. Multi-view VAE  

VAE используется для построения латентных представлений товаров и корзин.  

Вход модели состоит из нескольких представлений:  

- уровень товаров;  
- категория `lvl2`;  
- категория `lvl1`;  
- категория `lvl0`.  

Такой подход позволяет учитывать не только конкретные SKU, но и товарную иерархию.  

VAE обучается восстанавливать корзину по её частично скрытому представлению.   
На выходе модель формирует латентное пространство, которое используется как дополнительный источник признаков.  

---

### 4. Гибридная модель  

Гибридная модель объединяет EASE и VAE.  

Финальная схема:  

1. **EASE** генерирует top-K кандидатов.  
2. Для каждого кандидата строятся признаки:  
   - EASE score;  
   - VAE score;  
   - cosine similarity в VAE-пространстве;  
   - взаимодействия признаков;  
   - размер корзины;  
   - ранг кандидата по EASE.  
3. **Logistic Regression reranker** выполняет финальное ранжирование.  
4. Итоговый скор формируется как комбинация EASE score и вероятности reranker.  

Именно этот подход показал лучший результат по Recall@10.  

---

### 5. Добавление нового SKU без полного переобучения  

Отдельно реализован сценарий добавления нового товара.  

Для нового SKU:  

1. собираются чеки, в которых он встречается;  
2. считается co-occurrence с уже известными товарами;  
3. строится приближённый EASE-вектор;  
4. через VAE encoder получаются latent-представления корзин;  
5. VAE-эмбеддинг нового SKU получается через агрегацию этих представлений.   

Важно: старая матрица EASE и старое VAE-пространство не переобучаются.   

Это позволяет получать признаки для нового товара без полного retrain модели.

## Метрика качества  

В качестве основной метрики используется:  

```text
Recall@10
```

Схема оценки:  

1. из тестовой корзины случайно скрывается один товар;  
2. оставшиеся товары используются как контекст;  
3. модель формирует top-10 кандидатов;   
4. считается попадание скрытого товара в top-10.    

Формально:  

```text
Recall@10 = (1 / T) * Σ 1(i ∈ Top10(C))
```

где:  
   
- `T` — количество тестовых примеров;  
- `i` — скрытый товар;  
- `C` — контекст корзины;  
- `Top10(C)` — топ-10 предсказаний модели.  

Метрика выбрана потому, что работа исследует качество восстановления товара из корзины, а не полноценную персонализированную рекомендательную систему.

## Эксперименты  

В проекте сравниваются следующие подходы:  

| Модель | Описание |  
|---|---|  
| Popularity | рекомендация самых популярных товаров |  
| Word2Vec | baseline embedding-модель |  
| VAE | multi-view вариационный автоэнкодер |  
| EASE | линейная модель item-item связей |  
| Hybrid | EASE + VAE + Logistic Regression reranker |  

Финальная логика сравнения:  

```text
Hybrid > EASE > VAE > Popularity > Word2Vec  
```

Гибридная модель показала лучший результат, так как использует:   

- сильный item-item сигнал EASE;  
- латентные признаки VAE;  
- обучаемый reranker.

## Ограничения  

Основные ограничения проекта:  

1. **Масштабируемость EASE**

   Для большого числа SKU обращение матрицы может быть вычислительно дорогим.

2. **Cold start**

   Для нового SKU требуется некоторое количество чеков. Полностью холодный старт без данных не решается.

3. **Стохастичность Recall@10**

   Метрика зависит от случайного выбора скрываемого товара.

4. **Ограниченность признаков**

   В модели не используются цена, промо, сезонность и временная динамика.

5. **Разреженность данных**

   Редкие товары получают менее стабильные представления.
